# GROMACS MD Preparation

## Project Layout

```
PROJECT/
├── forcefield/          # Custom force field directory (e.g. ff99SBws-STQ.ff/)
├── prep/                # pdb2gmx output, solvated/ionised system
│   ├── system.gro       # Final starting coordinates
│   ├── topol.top        # Master topology
│   ├── index.ndx        # Custom index groups
│   └── posre_*.itp      # Position restraint files (generated by pdb2gmx)
├── mdp/                 # All .mdp parameter files
│   ├── em.mdp
│   ├── nvt.mdp
│   ├── npt.mdp
│   └── prod.mdp (or awh.mdp / hremd_rep0.mdp …)
├── scripts/             # Numbered shell scripts: 01_prep.sh, 02_em.sh …
├── runs/                # One subdirectory per stage
│   ├── em/
│   ├── nvt/
│   ├── npt/
│   └── prod/ (or awh/, hremd/)
└── analysis/            # Notebooks and processed data
```

## Environment and Defaults

| Parameter | Default | Notes |
|---|---|---|
| Force field | `ff99SBws-STQ` | Custom; lives in `forcefield/`; set `GMXLIB="$(pwd)/forcefield:$GMXLIB"` |
| Water model | TIP4P/2005 | GROMACS keyword: `tip4p2005`; solvate with `-cs tip4p` |
| Salt | 150 mM NaCl | `genion -pname NA -nname CL -neutral -conc 0.150` |
| Temperature | 300 K | Two coupling groups: `Protein Non-Protein` |
| Pressure | 1 bar | isotropic, `compressibility = 4.5e-5` |
| Timestep | 2 fs (`dt = 0.002`) | |
| Cutoffs | 1.0 nm | `rcoulomb = rvdw = 1.0` |
| Electrostatics | PME | `pme-order = 4`, `fourierspacing = 0.12` |
| VdW | Cut-off + DispCorr | `DispCorr = EnerPres` |
| Constraints | H-bonds | `constraint_algorithm = lincs`, `lincs-iter = 1`, `lincs-order = 4` |
| Box (IDP/peptide) | Dodecahedral | `-bt dodecahedron -d 1.0` |
| Box (globular) | Cubic | `-bt cubic -d 1.0` |

## Standard Workflow

```
PDB → pdb2gmx → editconf → solvate → genion → EM → NVT(posre) → NPT → production
```

---

## Stage 1: pdb2gmx

```bash
export GMXLIB="$(pwd)/forcefield:${GMXLIB:-}"

gmx pdb2gmx \
    -f protein.pdb \
    -o prep/protein_processed.gro \
    -p prep/topol.top \
    -i prep/posre.itp \
    -ff ff99SBws-STQ \
    -water tip4p2005 \
    -ignh
```

**Verify:** `posre.itp` created; no "missing atom" warnings; check residue numbering.

> For CHARMM36m use `-ff charmm36m -water tip3p`. Do **not** set `GMXLIB` unless using a custom FF directory.

---

## Stage 2: editconf (box)

```bash
# Peptide/IDP — dodecahedral box
gmx editconf \
    -f prep/protein_processed.gro \
    -o prep/boxed.gro \
    -bt dodecahedron \
    -d 1.0

# Globular protein — cubic box
gmx editconf \
    -f prep/protein_processed.gro \
    -o prep/boxed.gro \
    -bt cubic \
    -d 1.0
```

**Verify:** box vectors printed; volume reasonable for system size.

---

## Stage 3: solvate + genion

```bash
gmx solvate \
    -cp prep/boxed.gro \
    -cs tip4p \
    -o prep/solvated.gro \
    -p prep/topol.top

# Needs a dummy tpr — use em.mdp
gmx grompp \
    -f mdp/em.mdp \
    -c prep/solvated.gro \
    -p prep/topol.top \
    -o prep/ions.tpr \
    -maxwarn 2

echo "SOL" | gmx genion \
    -s prep/ions.tpr \
    -o prep/system.gro \
    -p prep/topol.top \
    -pname NA -nname CL \
    -neutral -conc 0.150
```

**Verify:** water/ion count in `prep/system.gro`; check `[ molecules ]` in `prep/topol.top`.

## Stage 4: Energy Minimisation (EM)

### mdp/em.mdp
```ini
; Energy minimization — steepest descent
; ff99SBws-STQ / TIP4P-2005

integrator               = steep
nsteps                   = 50000
emtol                    = 100.0       ; kJ/mol/nm
emstep                   = 0.01        ; nm

; Output
nstxout-compressed       = 500
nstlog                   = 500
nstenergy                = 500

; Neighbour list
cutoff-scheme            = Verlet
nstlist                  = 10
ns_type                  = grid
pbc                      = xyz

; Electrostatics
coulombtype              = PME
rcoulomb                 = 1.0
pme-order                = 4
fourierspacing           = 0.12

; Van der Waals
vdwtype                  = Cut-off
rvdw                     = 1.0
DispCorr                 = EnerPres
```

### Commands
```bash
mkdir -p runs/em
gmx grompp -f mdp/em.mdp -c prep/system.gro -p prep/topol.top \
           -n prep/index.ndx -o runs/em/em.tpr -maxwarn 2
gmx mdrun -v -deffnm runs/em/em -ntmpi 1
```

**Verify:**
```bash
echo "Potential" | gmx energy -f runs/em/em.edr -o runs/em/potential.xvg -xvg none
# Final potential should be large and negative; Fmax < emtol
```

## Stage 5: NVT Equilibration (with position restraints)

### mdp/nvt.mdp
```ini
; NVT equilibration — 200 ps, 300 K
; Position restraints on heavy atoms (POSRES defined in topology)
; ff99SBws-STQ / TIP4P-2005

define                   = -DPOSRES

integrator               = md
dt                       = 0.002        ; ps
nsteps                   = 100000       ; 200 ps
comm-mode                = Linear
nstcomm                  = 100

; Output
nstxout                  = 0
nstvout                  = 0
nstfout                  = 0
nstxout-compressed       = 5000         ; 10 ps
nstlog                   = 1000
nstenergy                = 1000

; Neighbour list
cutoff-scheme            = Verlet
nstlist                  = 10
pbc                      = xyz

; Electrostatics
coulombtype              = PME
rcoulomb                 = 1.0
pme-order                = 4
fourierspacing           = 0.12

; Van der Waals
vdwtype                  = Cut-off
rvdw                     = 1.0
DispCorr                 = EnerPres

; Bonds
constraints              = h-bonds
constraint_algorithm     = lincs
lincs-iter               = 1
lincs-order              = 4
continuation             = no

; Temperature — V-rescale
tcoupl                   = V-rescale
tc-grps                  = Protein Non-Protein
tau_t                    = 0.1    0.1
ref_t                    = 300    300

; No pressure coupling in NVT
pcoupl                   = no

; Generate velocities
gen_vel                  = yes
gen_temp                 = 300
gen_seed                 = -1
```

### Commands
```bash
mkdir -p runs/nvt
gmx grompp -f mdp/nvt.mdp -c runs/em/em.gro -p prep/topol.top \
           -r runs/em/em.gro -n prep/index.ndx -o runs/nvt/nvt.tpr -maxwarn 2
gmx mdrun -v -deffnm runs/nvt/nvt -ntmpi 1
```

**Verify:**
```bash
echo "Temperature" | gmx energy -f runs/nvt/nvt.edr -o runs/nvt/temperature.xvg -xvg none
# Temperature should plateau at ~300 K in the second half of the run
```

## Stage 6: NPT Equilibration

### mdp/npt.mdp
```ini
; NPT equilibration — 1 ns, 300 K, 1 bar
; ff99SBws-STQ / TIP4P-2005

integrator               = md
dt                       = 0.002        ; ps
nsteps                   = 500000       ; 1 ns
comm-mode                = Linear
nstcomm                  = 100

; Output
nstxout                  = 0
nstvout                  = 0
nstfout                  = 0
nstxout-compressed       = 5000         ; 10 ps
nstlog                   = 1000
nstenergy                = 1000

; Neighbour list
cutoff-scheme            = Verlet
nstlist                  = 10
pbc                      = xyz

; Electrostatics
coulombtype              = PME
rcoulomb                 = 1.0
pme-order                = 4
fourierspacing           = 0.12

; Van der Waals
vdwtype                  = Cut-off
rvdw                     = 1.0
DispCorr                 = EnerPres

; Bonds
constraints              = h-bonds
constraint_algorithm     = lincs
lincs-iter               = 1
lincs-order              = 4
continuation             = yes          ; velocities from NVT checkpoint

; Temperature — V-rescale
tcoupl                   = V-rescale
tc-grps                  = Protein Non-Protein
tau_t                    = 0.1    0.1
ref_t                    = 300    300

; Pressure — Parrinello-Rahman
pcoupl                   = Parrinello-Rahman
pcoupltype               = isotropic
tau_p                    = 2.0          ; ps
ref_p                    = 1.0          ; bar
compressibility          = 4.5e-5       ; bar⁻¹
refcoord-scaling         = com

gen_vel                  = no
```

> **Note:** For early equilibration of poorly equilibrated systems, substitute `Berendsen` for `Parrinello-Rahman` (`tau_p = 2.0`). Switch to Parrinello-Rahman for any production run.

### Commands
```bash
mkdir -p runs/npt
gmx grompp -f mdp/npt.mdp -c runs/nvt/nvt.gro -t runs/nvt/nvt.cpt \
           -p prep/topol.top -n prep/index.ndx -o runs/npt/npt.tpr -maxwarn 2
gmx mdrun -v -deffnm runs/npt/npt -ntmpi 1
```

**Verify:**
```bash
echo "Density\nPressure" | gmx energy -f runs/npt/npt.edr \
    -o runs/npt/density.xvg -xvg none
# Density should stabilise at ~1000 kg/m³; pressure fluctuates around 1 bar
```

## Stage 7: Production MD

### NVT production (preferred for IDPs — Langevin dynamics)

### mdp/prod_nvt.mdp
```ini
; Production NVT — sd (Langevin) integrator
; ff99SBws-STQ / TIP4P-2005

integrator               = sd
dt                       = 0.002        ; ps
nsteps                   = 50000000     ; 100 ns (adjust as needed)
comm-mode                = Linear
nstcomm                  = 100

; Output
nstxout                  = 0
nstvout                  = 0
nstfout                  = 0
nstxout-compressed       = 5000         ; 10 ps
nstlog                   = 5000
nstenergy                = 5000
compressed-x-grps        = non-Water

; Neighbour list
cutoff-scheme            = Verlet
nstlist                  = 10
pbc                      = xyz

; Electrostatics
coulombtype              = PME
rcoulomb                 = 1.0
pme-order                = 4
fourierspacing           = 0.12

; Van der Waals
vdwtype                  = Cut-off
rvdw                     = 1.0
DispCorr                 = EnerPres

; Bonds
constraints              = h-bonds
constraint_algorithm     = lincs
lincs-iter               = 1
lincs-order              = 4
continuation             = yes

; Temperature — sd uses its own thermostat (ld-seed = -1 for random)
tc-grps                  = Protein Non-Protein
tau_t                    = 1.0    1.0
ref_t                    = 300    300

; No pressure coupling (NVT production)
pcoupl                   = no

gen_vel                  = no
```

### Commands
```bash
mkdir -p runs/prod
gmx grompp -f mdp/prod_nvt.mdp -c runs/npt/npt.gro -t runs/npt/npt.cpt \
           -p prep/topol.top -n prep/index.ndx -o runs/prod/prod.tpr -maxwarn 2
gmx mdrun -v -deffnm runs/prod/prod -ntmpi 1 -ntomp 4 -pme gpu
```

**Verify:**
```bash
gmx check -f runs/prod/prod.xtc 2>&1 | grep "^Last frame"
# Should show non-zero frame count
```

## SLURM Templates

### Single GPU (single run)

```bash
#!/bin/bash
#SBATCH --job-name=JOBNAME
#SBATCH --partition=PARTITION
#SBATCH --nodes=1
#SBATCH --ntasks=1
#SBATCH --cpus-per-task=8
#SBATCH --gres=gpu:1
#SBATCH --time=24:00:00
#SBATCH --output=slurm_%j.out

module load gromacs/2024.1-gpu   # adjust to your cluster module

TPR="runs/prod/prod.tpr"
DEFFNM="runs/prod/prod"

gmx mdrun \
    -v \
    -s "${TPR}" \
    -deffnm "${DEFFNM}" \
    -ntmpi 1 \
    -ntomp "${SLURM_CPUS_PER_TASK}" \
    -pme gpu \
    -bonded gpu
```

### Multi-replica (H-REMD / AWH multi-sim)

```bash
#!/bin/bash
#SBATCH --job-name=HREMD
#SBATCH --partition=PARTITION
#SBATCH --nodes=1
#SBATCH --ntasks=NREPLICAS        # e.g. 8 for 8 replicas
#SBATCH --cpus-per-task=4
#SBATCH --gres=gpu:NREPLICAS
#SBATCH --time=48:00:00
#SBATCH --output=slurm_%j.out

module load gromacs/2024.1-mpi-gpu

NREP=NREPLICAS
PREFIX="runs/hremd/hremd_rep"

mpirun -np ${NREP} gmx_mpi mdrun \
    -v \
    -s "${PREFIX}" \
    -deffnm "${PREFIX}" \
    -multi ${NREP} \
    -replex 1000 \
    -ntomp "${SLURM_CPUS_PER_TASK}"
```

## Common Variations

### Force Field Table

> **Always prefer the most recent validated force field for the target system. Do not use charmm22* or charmm27 for new simulations.**

| FF name (GROMACS) | Family | Status | Notes |
|---|---|---|---|
| `ff99SBws-STQ` | AMBER | **Preferred** | IDPs/peptides; custom, lives in `forcefield/`; pair with TIP4P/2005 |
| `ff19SB` | AMBER | **Preferred** | Folded + disordered; pair with OPC water |
| `a99SBdisp` | AMBER | **Preferred** | IDPs; pair with TIP4P-D |
| `charmm36m` | CHARMM | **Preferred** | IDPs and folded; pair with CHARMM-modified TIP3P |
| `amber99sb-ildn*` | AMBER | Acceptable (older) | Folded proteins only; superseded by ff19SB |
| `amber99sb*` | AMBER | Acceptable (older) | Missing ILDN corrections; use only when compatibility required |
| `charmm22*` / `charmm27` | CHARMM | **Discouraged** | Superseded by charmm36m |

### Water Model Table

| Water | GROMACS keyword | Solvate `-cs` | Compatible FF |
|---|---|---|---|
| TIP4P/2005 | `tip4p2005` | `tip4p` | ff99SBws-STQ |
| TIP3P | `tip3p` | `spc216` | AMBER, CHARMM |
| SPC/E | `spce` | `spc216` | AMBER |
| CHARMM TIP3P | `tip3p` | `spc216` | CHARMM36m |

### Box Shape Guidance

| System | Box shape | `-d` margin | Rationale |
|---|---|---|---|
| Peptide / IDP | dodecahedron | 1.0 nm | Standard rule of thumb; keeps box compact |
| Globular protein | cubic or dodecahedron | 1.0 nm | Standard rule of thumb |
| Membrane protein | hexagonal / slab | varies | Requires lipid setup |

## Special Cases

### Ligands / Custom Molecules (ACPYPE → GROMACS)

```bash
# 1. Parametrise with ACPYPE (GAFF2 force field)
acpype -i ligand.mol2 -c bcc -n 0 -a gaff2

# 2. Copy generated files
cp ligand.acpype/ligand_GMX.itp prep/ligand.itp
cp ligand.acpype/ligand_GMX.gro prep/ligand.gro

# 3. In topol.top — add before [ system ]
#include "ligand.itp"

# 4. In [ molecules ] section, add ligand line
# LIG   1

# 5. Combine coordinates
gmx insert-molecules -f prep/protein_processed.gro \
    -ci prep/ligand.gro -nmol 1 -o prep/complex.gro -radius 0.3

# 6. Continue with editconf → solvate → genion as normal
```

### AWH Free Energy (COM distance)

Build on NPT-equilibrated system. Requires index groups for the two molecules/domains.

**Additional mdp sections** (add to npt.mdp base):

```ini
; Pull code — COM distance as AWH external-potential coordinate
pull                     = yes
pull-ngroups             = 2
pull-group1-name         = MOL_A       ; must exist in index.ndx
pull-group2-name         = MOL_B
pull-ncoords             = 1
pull-coord1-type         = external-potential
pull-coord1-potential-provider = AWH
pull-coord1-geometry     = distance
pull-coord1-groups       = 1 2
pull-coord1-start        = no
pull-coord1-dim          = Y Y Y       ; 3-D scalar distance

; AWH
awh                      = yes
awh-potential            = convolved
awh-share-multisim       = no
awh-seed                 = -1
awh-nstout               = 5000        ; write bias every 10 ps (multiple of nstenergy)

; AWH bias 1
awh-nbias                = 1
awh1-error-init          = 10.0        ; kJ/mol
awh1-growth              = exp-linear
awh1-equilibrate-histogram = yes
awh1-target              = constant
awh1-ndim                = 1

; Dimension 1
awh1-dim1-coord-index    = 1
awh1-dim1-start          = START_NM    ; nm — closest approach
awh1-dim1-end            = END_NM      ; nm — fully separated
awh1-dim1-force-constant = 4000        ; kJ/mol/nm²
awh1-dim1-diffusion      = 5e-5        ; nm²/ps (initial estimate)
```

**Index group requirement:**
```bash
gmx make_ndx -f prep/system.gro -o prep/index.ndx << 'EOF'
r FIRST_RESID-LAST_RESID_A
name N MOL_A
r FIRST_RESID-LAST_RESID_B
name N+1 MOL_B
q
EOF
```

**AWH run command:**
```bash
gmx grompp -f mdp/awh.mdp -c runs/npt/npt.gro -t runs/npt/npt.cpt \
    -p prep/topol.top -n prep/index.ndx -o runs/awh/awh.tpr -maxwarn 2
gmx mdrun -v -deffnm runs/awh/awh -ntmpi 1 -ntomp 4 -pme gpu
```

**Extract PMF:**
```bash
gmx awh -s runs/awh/awh.tpr -f runs/awh/awh.edr -o analysis/pmf.xvg -more
```

### H-REMD (Hamiltonian Replica Exchange)

Uses free-energy perturbation with vdw_lambdas to scale water–solute interactions.

**Rep-0 mdp template** (`mdp/hremd_rep0.mdp`):

```ini
integrator               = sd
dt                       = 0.002
nsteps                   = 5000000      ; 10 ns per replica
nstlog                   = 1000
nstenergy                = 1000
nstxout-compressed       = 1000
compressed-x-grps        = non-Water
continuation             = yes
constraint_algorithm     = lincs
constraints              = h-bonds
lincs-iter               = 1
lincs-order              = 4
cutoff-scheme            = Verlet
nstlist                  = 10
coulombtype              = PME
rcoulomb                 = 1.0
rvdw                     = 1.0
DispCorr                 = EnerPres
tc-grps                  = Protein Non-Protein
tau_t                    = 1.0    1.0
ref_t                    = 300    300
pcoupl                   = no
gen_vel                  = no

; H-REMD settings
free-energy              = yes
init-lambda-state        = 0            ; replica index (0-based)
nstdhdl                  = 1000
vdw-lambdas              = 0.00 0.15 0.30 0.45 0.60 0.75 0.90 1.00
```

**Generate per-replica mdp files:**
```bash
NREP=8
for i in $(seq 1 $((NREP-1))); do
    awk -v i=$i '/init-lambda-state/{print "init-lambda-state        = " i; next}{print}' \
        mdp/hremd_rep0.mdp > mdp/hremd_rep${i}.mdp
done
```

**Prepare tpr files (one per replica):**
```bash
for r in $(seq 0 $((NREP-1))); do
    gmx grompp -f mdp/hremd_rep${r}.mdp \
        -c runs/npt/npt.gro -t runs/npt/npt.cpt \
        -p prep/topol.top -n prep/index.ndx \
        -o runs/hremd/hremd_rep${r}.tpr -maxwarn 1
done
```

**Run (MPI required):**
```bash
mpirun -np ${NREP} gmx_mpi mdrun \
    -s runs/hremd/hremd_rep \
    -deffnm runs/hremd/hremd_rep \
    -multi ${NREP} \
    -replex 1000 \
    -ntomp 4
```

**Demux trajectories** (after run):
```bash
# requires demux.pl from GROMACS tools or gmx trjcat
demux.pl runs/hremd/hremd_rep0.log
```

## Force-Field-Specific MDP Requirements

**These settings must be verified for every mdp file generated or reviewed.**

### Cutoffs and VdW treatment

| Force field | `rcoulomb` | `rvdw` | `vdwtype` | `vdw-modifier` | `rvdw-switch` | `DispCorr` |
|---|---|---|---|---|---|---|
| AMBER (ff99SBws-STQ, ff19SB, a99SBdisp, amber99sb-ildn*) | 1.0 | 1.0 | Cut-off | — | — | EnerPres |
| CHARMM36m | 1.2 | 1.2 | Cut-off | force-switch | 1.0 | **no** (omit or set `no`) |

### Constraints

| Force field | `constraints` | `constraint_algorithm` | `lincs-order` | Notes |
|---|---|---|---|---|
| AMBER family | h-bonds | lincs | 4 | Standard; 2 fs timestep safe |
| CHARMM36m | h-bonds | lincs | 4 | Use all-bonds only if dt > 2 fs |

### Pre-grompp validation checklist

```
BEFORE running grompp, verify:
[ ] rcoulomb and rvdw match the FF table above
[ ] vdw-modifier = force-switch and rvdw-switch = 1.0 if CHARMM36m; absent otherwise
[ ] DispCorr = EnerPres for AMBER; absent (or = no) for CHARMM36m
[ ] constraints and lincs settings match FF table
[ ] Water model compatible: TIP4P/2005 → ff99SBws-STQ; OPC → ff19SB; TIP4P-D → a99SBdisp; CHARMM TIP3P → charmm36m
[ ] coulombtype = PME for all production runs
```

## Common Errors and Fixes

| Error | Cause | Fix |
|---|---|---|
| `Fatal error: Inconsistent box` | box too small after solvate | increase `-d` in editconf |
| `WARNING: 1-4 interaction ... exceeds cutoff` | rvdw < 1.0 or wrong FF | set `rvdw = 1.0`; check FF parameters |
| `LINCS warning: step X` (many) | constraints not satisfied | reduce `dt` temporarily; check for clashes after genion |
| `Pressure coupling with Berendsen → Parrinello-Rahman instability` | switching during production | run at least 1 ns Berendsen NPT before switching |
| `AWH: histogram not equilibrated` | run too short | increase `nsteps`; AWH needs thousands of round-trips |
| `mpirun: cannot connect to PMI` for H-REMD | wrong MPI build | use `gmx_mpi` binary, compiled with MPI; check `module list` |
| `segfault in pme_gpu` | GPU driver / GROMACS version mismatch | run with `-pme cpu` to test; update drivers or GROMACS |
